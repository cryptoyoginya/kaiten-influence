# Media Playbook: закупка, атрибуция, портфель

> Перфоманс-правила поверх конвейера креативов. Кто за что отвечает:
> [CREO-RUNBOOK](CREO-RUNBOOK.md) — качество креатива, этот файл — **экономика решения о покупке
> и замыкание петли фактом**. Принцип тот же, что во всём репо: чего нет в данных, того нет в решении.

## 1 · Юнит-экономика закупки (до сделки)

Каждое размещение до оплаты должно иметь три числа в карточке:

| Метрика | Как считается | Правило |
|---|---|---|
| **Прогноз CPV** | цена / прогноз охвата поста (не подписчики!) | считать по медиане просмотров последних 10 постов (TGStat/ручками), не со слов продавца |
| **Прогноз CPL** | CPV / ожидаемый CTR × конверсия лендинга | первые сделки — по гипотезе, дальше по своим фактам |
| **Допустимый CAC** | из экономики Kaiten (LTV тарифа) | потолок, выше которого сделка не согласуется |

⚠️ **Бенчмарки-гипотезы до первых фактов** (заменить своими цифрами после 5+ размещений):
посевы Telegram B2B-SaaS RU — CPV 1.5–4 ₽ норм, >6 ₽ дорого; CTR поста 0.5–1.5%;
CPL 1 500–4 000 ₽ терпимо для B2B. Каждый факт уточняет эти пороги — см. §5.

**Красные флаги закупки** (любой = торг или отказ):
- цена за пост выше медианы ниши при том же охвате;
- ER канала падает 3 месяца подряд / рекламный пост чаще, чем 1 из 3;
- просмотры поста аномально ровные (накрутка) — смотреть распределение, не среднее;
- канал уже «сжёг» наш жанр чужой рекламой за последние 2–3 недели.

## 2 · Переговорный чек-лист (каждая сделка)

- [ ] Скидка от прайса (спросить всегда; молчание = −10–20% недополучено)
- [ ] **Пакет**: 2-е размещение через 4–6 недель со скидкой 20–30% — дешевле ретест победителя
- [ ] Закреп 24 ч / без удаления N дней — прописать в договоре
- [ ] **Право переиспользовать креатив** (whitelisting: другие каналы, таргет, сайт) — бесплатно сейчас, дорого потом
- [ ] Статистика поста от автора на T+72ч (просмотры/пересылки/реакции) — как условие сделки
- [ ] Дата и время выхода: вечер вт–чт для B2B, не в дни больших инфоповодов

## 3 · Атрибуция: стандарт UTM и точки замера

**Единый шаблон UTM** (без него пост не публикуется — добавить в пред-публикационный чек):

```
utm_source=telegram
utm_medium=influencer
utm_campaign=<channel-slug>            # sveta-duchak, pm-god…
utm_content=<set>-<variant>            # 2026w2-z1
```

- Один вариант = один `utm_content` → A/B-прогноз из ab-plan проверяется фактом напрямую.
- Где возможно — **промокод канала** (второй канал атрибуции, ловит отложенные конверсии и пересылки).
- Ссылку в посте укорачивать честно (kaiten.ru/... с UTM), не терять параметры на редиректах лендинга.

**Точки замера факта:** T+24ч (первая волна) и T+72ч (основной замер) → §4.

## 4 · Ритуал факта (замыкание петли) — ОБЯЗАТЕЛЬНО

Скоринг без факта — вкусовщина с лишними шагами. Поэтому у каждого опубликованного размещения
есть **долг по факту**, и он проверяется скриптом:

```
python3 scripts/check-facts.py        # список долгов: опубликовано, а факта нет
```

Ритуал:
1. **T+24ч** — просмотры, реакции, пересылки, комментарии (скрин статистики от автора в карточку).
2. **T+72ч** — то же + клики по UTM, регистрации → заполняются `result.*` в Результатах
   и `facts` в `ab-plan.json` набора.
3. **Калибровка** (шаг 9 CREO-RUNBOOK): сверить факт с предсказаниями ab-plan —
   какие гипотезы сошлись, какие нет → правка портретов персон / голос-профиля / весов,
   вердикт в `calibration`. Предсказания задним числом не переписываются.
4. Уроки (`lessons`) — 3 строки: что сработало, что нет, что меняем в следующем сете.

## 5 · Портфель и бюджет

- **70 / 20 / 10**: 70% бюджета — каналы с подтверждённым CPV ниже своей медианы,
  20% — новые каналы проверенных ниш, 10% — эксперименты (новые ниши/форматы).
  До первых фактов всё считается «20%» — поэтому факты в приоритете.
- **Ретест победителя**: канал с CPV ниже медианы получает повторное размещение
  через 4–6 недель (другой вариант сета — уже написан и отскорен, ноль доп. затрат).
- **Частотность**: один канал — не чаще 1 раза в 6–8 недель (выгорание аудитории).
- **Медиана обновляется** после каждого факта: `цена/факт-охват` по всем вышедшим —
  это и есть живой бенчмарк вместо гипотез §1.

## 6 · Скоринг канала (до попадания в спринт)

Быстрый фильтр бэклога (0–10, порог в спринт ≥ 6):

| Ось | Вес | Сигнал |
|---|---|---|
| ICP-fit | ×3 | аудитория канала = роли из icp-kaiten.md |
| Экономика | ×3 | прогноз CPV против медианы сегмента |
| Живость | ×2 | ER, динамика подписок, реальные комментарии |
| Ad-load | ×1 | доля рекламы в ленте (реже = лучше) |
| Brand safety | ×1 | тональность канала не конфликтует с Kaiten |

## Связь с конвейером

```
Бэклог (скоринг канала §6) → закупка (юнит-экономика §1 + торг §2)
  → /creo (сет+скоринг+валидатор) → публикация (UTM §3)
  → факт T+24/T+72 (§4, check-facts.py) → калибровка → медиана §5 → снова бэклог
```
