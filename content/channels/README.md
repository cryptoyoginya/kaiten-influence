# Каналы: стандарт папки инфлюенсера

> Каждый канал, с которым работаем (и каждый новый из бэклога при переносе
> в спринт), получает свою папку по этому стандарту. Скоринг-пайплайн
> ([SCORING-PIPELINE.md](../../docs/SCORING-PIPELINE.md)) читает эти файлы
> на этапе интейка; чего нет — честно помечается в отчёте прогона.

## Структура

```
content/channels/<slug>/
  channel.md         # паспорт: автор, аудитория, сегмент-панель, экономика, оферта, чек-лист данных
  voice-profile.md   # голос автора; черновик до занесения реальных постов, потом подтверждённый
  real-posts.md      # корпус 5–7 реальных постов канала (появляется по мере занесения)
  sets/<set-id>/
    set.md           # тексты вариантов + карта осей разнообразия + статусы правок
    ab-plan.json     # предсказания до публикации + факт + калибровка
```

## Правила

1. **Сегмент-панель обязательна.** В паспорте канала указывается панель ЦА
   из `content/segments/` (editorial, it-dev, …). Нет панели сегмента —
   сначала собрать по инструкции в конце panel.md.
2. **Голос до текстов.** Без реальных постов voice-profile остаётся черновиком,
   а voice-match в прогоне помечается «по черновому профилю» — это видно
   в отчёте и снижает доверие к Craft.
3. **Экономика из карточки.** Цена, подписчики, охват берутся из карточки
   размещения в спринте и дублируются в паспорт при заведении.
4. **Один паспорт — много сетов.** Данные канала живут в корне, каждый
   рекламный сет — отдельной папкой в sets/.
