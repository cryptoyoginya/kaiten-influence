# Углы из KB продукта для «Импортозамещение в ИТ» — без опоры на Jira

> Запрос акционеров: не делать Jira единственным/центральным углом. Ниже — все релевантные заходы из KB
> ([kaiten-product-facts.md](../../../../content_zavod/wiki/references/kaiten-product-facts.md) §, ICP [icp-kaiten.md](../../references/icp-kaiten.md), company-overview, платформенный лендинг). У каждого — факты KB, ICP-роль, есть ли Jira.

## A. Суверенитет и регуляторика (Jira не нужна) 🟢
- **Факты:** Единый реестр российского ПО · аккредитация ИТ-компании · сертификат **«Промтехстандарт»** · соответствие **152-ФЗ**, Минцифры · проверки **ФСТЭК** · данные хранятся в России.
- **ICP:** CIO банка (регуляторы финсектора, ЦБ/Роскомнадзор), замминистра (госзакупки, ФСТЭК), Head of Import Substitution.
- **Пруф:** «Сколково выбирает Kaiten» (репортаж ТВЦ об импортозамещении ПО).

## B. On-premise / свой сервер (Jira не нужна) 🟢
- **Факты:** установка на свой сервер (Docker-контейнер) · on-premise развёртывание · гибрид · данные не покидают контур. Для госа облако запрещено.
- **ICP:** замминистра («нет on-prem — отказ в закупке»), CIO банка, ИБ-директор.

## C. Непрерывность и риск иностранного ПО (без имени Jira) 🟢
- **Факты/боли ICP:** иностранное ПО не обновляется → уязвимости ИБ; риск отключения → простой, срыв релизов/контрактов. Метрики: **uptime %**, дни простоя → 0, инциденты ИБ → 0.
- **ICP:** Head of Import Substitution («риск штрафов/прекращения поддержки → паралич»), Head of Engineering, CIO.
- Формулировать как «зависимость от зарубежного ПО», не «уход Jira».

## D. Функциональная полнота — 9 модулей (Jira не нужна) 🟢
- **Факты:** 9 модулей закрывают потребности целиком: канбан/скрам, Гант + ресурсное планирование, портфель проектов, Service Desk, учёт времени, автоматизации, документы/база знаний, управление группами, USM. «Заменяет несколько инструментов».
- **ICP:** Head of Import Substitution («потребности удовлетворены в полном объёме»), PMO.
- Заход: «импортозамещение без потери функциональности».

## E. Замена не только трекера: Confluence / Notion / Excel / MS Project 🟢
- **Факты (§34):** документы и база знаний вместо Confluence/Notion; Гант вместо MS Project; «от таблицы к доске» вместо Excel.
- **ICP:** COO/PMO, CDO (сотрудники «в Excel/на бумаге»).
- Уводит фокус с Jira — импортозамещаем весь стек, не один трекер.

## F. Доверие крупных и масштаб (Jira не нужна) 🟢
- **Факты (§26):** 100 000+ компаний; Сбер, S7, X5, Ростелеком, Норникель. Кейсы: hopper-it (импортозамещение), НИИ эпидемиологии (гос), федеральный проект «Иксора».
- **ICP:** CIO, замминистра («потянет на масштабе»).
- ⚠️ Кейс X5 — это переезд с Jira; для «без Jira» брать hopper-it / НИИ / Сколково / федеральные.

## G. Экономика суверенитета (Jira не нужна) 🟢
- **Факты:** на **30–50% дешевле** зарубежных аналогов · модульная цена «платите за то, что нужно» · безлимитный **Free** по участникам.
- **ICP:** CFO/CIO, средний бизнес.

## H. Интеграции и API (без Jira) 🟢
- **Факты:** открытый REST API, интеграционная шина, Telegram/Slack; связка с GitHub/GitLab/CI (ICP Крупная IT).
- **ICP:** Head of Engineering («трекер не интегрируется с репозиторием/CI → ручная синхронизация»).

## I. Русский интерфейс, порог входа, adoption (Jira не нужна) 🟢
- **Факты:** интерфейс и поддержка на русском · интуитивность · постепенное внедрение модулей.
- **ICP:** замминистра («чуждый софт, язык»), CDO (user adoption rate, сопротивление сотрудников).

## J. Миграция из многих систем (Jira — один из, не центр) 🟡
- **Факты (§47):** импорт из Trello, Asana, Notion, ClickUp, Jira + универсальный CSV/JSON.
- Если нужен миграционный угол — подавать как «импорт из любой зарубежной системы», Jira не выделять.

## Как это меняет сет (ребаланс без Jira)

Текущий сет слишком Jira-центричен (победитель «Уход Atlassian», c1/c3/c4 про Jira). Ребаланс по углам:
- **Победитель →** угол **C+A** (непрерывность + суверенитет): «Иностранный трекер не обновляется — это риск ИБ и простоя» → отечественный Kaiten, реестр/сертификат/on-prem. Без имени Jira.
- **З2 →** угол **D** (9 модулей, полнота импортозамещения).
- **З3 →** угол **F/A** (кейсы + сертификат: hopper-it, Сколково, НИИ — не X5).
- **З4 →** угол **B** (on-prem/152-ФЗ для банков и госа).
- **З5 →** угол **G/E** (экономика + замена всего стека Confluence/Excel).

Визуалы: c2 (реестр) и c5 (on-prem) — уже без Jira, оставить. c1/c3/c4 — переделать под углы C/D/F (сервер+щит, 9 модулей, логотипы крупных без «замены Jira»).
